Elixir
------
Built on top of Erlang & its runtime
Runs on BEAM VM
Pheonix web framework
Nerves IoT framework


Why elixir?
- Backend language without mutable state
- Modern toolset
- Docs as a first class citizen
- High resilience & uptime

Don't use elixir if you:
- want static types
- want a fully matured ecosystem
- prefer OOP


String concatenation: "2" <> "b"
&& & || expect anything
and & or expect a boolean type comparison

Lists are not arrays - they're linked lists. Pushing is O(n), prepending is O(1)
Maps are objects as they are in JavaScript
Tuples should be treated as immutable collections grouping data together. Example: coordinates

Gotcha: you can check types in Elixir

Modules: a way to group functions
Alias: a way to call a module shorthand

Anonymous functions: hello = fn name -> "hello #{name}" end

Pattern matching

= match operator
[a, 3, true] = [1, 3, true]
customer1 = %{name: "something", age: 25}
%{age: anything} = age_map2 = customer1

^ pin operator
Used for matching only, not reassigning
a = 3
a = 2
a will be 2 now

^a = 3 will throw
^a = 2 will not

Key thing with pattern matching is the ability to replace if/else statements with functions

Pipe operator

username = name |> String.trim() |> String.downcase()
